Supadense uses a large language model (LLM) to power ingestion, extraction, and chat. You can choose which model to use and configure it through the app settings or your `opencode.json` config file.

---

## Supported providers

Supadense works with the major AI providers out of the box:

| Provider | Examples |
|----------|---------|
| **Anthropic** | Claude Sonnet 4, Claude Opus 4 |
| **OpenAI** | GPT-4o, GPT-4.1, o3 |
| **Google** | Gemini 2.5 Pro, Gemini 2.5 Flash |
| **OpenRouter** | Any model via OpenRouter |

To connect a provider, open **Settings → Providers** and enter your API key.

---

## Recommended models

Supadense requires a model that is capable of both **long-context understanding** (for reading full articles, papers, and transcripts) and **reliable tool use** (for writing structured KB placements).

The following models are recommended:

| Model | Provider | Best for |
|-------|----------|---------|
| **Claude Sonnet 4** | Anthropic | Best overall — fast, accurate tool use |
| **Claude Opus 4** | Anthropic | Highest quality extractions, slower |
| **GPT-4.1** | OpenAI | Strong tool use, good at structured output |
| **Gemini 2.5 Pro** | Google | Long context, good for large PDFs |

:::tip
Claude Sonnet 4 is the default and what Supadense is optimized for. If you're unsure, start here.
:::

## Changing the model

### From the app

Click the **model name** shown in the session header to open the model picker. Select any model from your connected providers.

The selected model is saved per-session and remembered for new sessions.

### From the config

To set a default model for all sessions, add it to your `opencode.json`:

```json title="opencode.json"
{
  "$schema": "https://supadense.com/config.json",
  "model": "anthropic/claude-sonnet-4-5"
}
```

The format is `provider_id/model_id`. You can find the exact IDs in the model picker.

## The KB Curator model

Supadense runs a separate background agent — the **KB Curator** — when you add a resource via `/memorize`. The curator is responsible for reading the resource and placing extracted content into your wiki.

By default, the curator uses the same model as your main session. You can set a different model for it in `opencode.json` to balance cost and quality:

```json title="opencode.json"
{
  "$schema": "https://supadense.com/config.json",
  "agents": {
    "kb-curator": {
      "model": "anthropic/claude-haiku-4-5"
    }
  }
}
```

Using a faster, cheaper model for the curator (like Claude Haiku) keeps ingestion costs low while reserving a more capable model for your main chat sessions.

---

## Context window

The ingestion pipeline sends up to **12,000 characters** of raw resource content to the curator per run. For very long resources (long PDFs, full-length YouTube transcripts), the content is truncated at this limit before being passed to the model.

Most models with a 32k+ context window handle this comfortably.

---

## Cost

Every `/memorize` call runs one curator session — typically **1–3 LLM calls** depending on how many sections the resource is placed into. A typical ingestion costs a few cents or less with Claude Sonnet 4.

Chat responses in the KB session (answering questions from your KB) do not call the curator and cost only what a normal chat message costs.
