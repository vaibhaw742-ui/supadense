[**Supadense**](/) is an open source AI coding agent. It's available across:
terminal (TUI), desktop app (Mac/Windows), web app, and IDE extensions (VS Code, JetBrains).

---

## Prerequisites

To use Supadense in your terminal, you'll need a modern terminal emulator such as:

- [WezTerm](https://wezfurlong.org/wezterm/) (cross-platform)
- [Alacritty](https://github.com/alacritty/alacritty) (cross-platform)
- [Ghostty](https://github.com/ghostty-org/ghostty) (Linux and macOS)
- [Kitty](https://sw.kovidgoyal.net/kitty/) (Linux and macOS)

You'll also need an API key from at least one LLM provider, or you can use [Supadense Zen](/docs/zen) which provides a curated set of models.

## Configure

Run Supadense and use the `/connect` command to configure your LLM provider. You'll be directed to
`{console}` to authenticate and then prompted to enter your API key.

:::tip
Use [Supadense Zen](/docs/zen) for a curated set of models with tested defaults.
:::

## Initialize

Navigate to your project directory and run Supadense.

```bash
cd /path/to/my/project
opencode
```

Then run the `/init` command to have Supadense analyze your project and create an `AGENTS.md` file.
This file gives context to the AI on how your project is structured. Commit it to version control.

## Usage

### Ask questions

Use the `@` key to reference specific files in your project.

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### Add features

Use **Plan Mode** (hit `Tab`) to have Supadense layout a plan before writing any code. You can provide
feedback on the plan before it starts building.

- Hit `Tab` to enter **Plan Mode** before submitting your message
- Ask for a detailed plan of what you want to build
- Include images by dragging and dropping them into the chat
- Hit `Tab` again to switch back to **Build Mode** once you're happy with the plan

### Make changes

When asking Supadense to make changes, providing as much context as possible will help it make
the right changes. You can reference specific files with `@`.

### Undo changes

Supadense tracks all the changes it makes so you can undo them. Use `/undo` to undo the last set of
changes or `/redo` to redo them. You can undo multiple times to go back further.
