Remove a category from the knowledge base.

Arguments: $ARGUMENTS
(Provide the category name or slug to remove, e.g. "Agents" or "agents")

Steps:
1. Call `kb_workspace_init` to get the workspace.
2. List the current categories from the workspace to match $ARGUMENTS to the correct category_slug.
3. Look up the category's wiki pages to determine its folder path (the overview page's directory).
4. Ask the user for explicit confirmation before proceeding. Show them exactly what will be deleted:

   "⚠️ This will permanently delete:
      📁 <folder_path>/
         overview.md
         key-concepts.md
         (and any other sections inside)
   All DB records and resource placements for this category will also be removed.
   Type **yes** to confirm, or anything else to cancel."

   Wait for the user's reply. Only proceed if they confirm with "yes" (case-insensitive). Otherwise, stop and tell them the operation was cancelled.

5. Once confirmed, call `kb_category_manage` with action `remove_category` and the matched category_slug.
6. Confirm the category has been removed.
